# Accessibility

Accessibility requirements apply before a screen is considered done.

## Practical Rules

- Text and interactive controls must meet WCAG AA contrast for their size.
- Every interactive element must be reachable and usable with a keyboard.
- Visible focus states must exist for buttons, links and form controls.
- Do not rely only on color to communicate loading, success, warning or error states.
- Use semantic HTML before custom ARIA.
- Error messages should explain the problem and the recovery path.
