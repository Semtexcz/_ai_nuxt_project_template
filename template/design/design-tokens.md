# Design Tokens

Design tokens document shared visual decisions. Keep tokens small until the product has enough screens to justify expansion.

## Color

- `color.text.primary`: primary readable text.
- `color.text.muted`: secondary text and helper copy.
- `color.background.page`: default page background.
- `color.background.surface`: panels, dialogs and repeated items.
- `color.border.default`: quiet separation between surfaces.
- `color.action.primary`: primary interactive action.
- `color.action.primaryHover`: primary action hover state.
- `color.feedback.error`: destructive or failed state.
- `color.feedback.success`: completed or healthy state.

## Typography

- Use system fonts unless brand requirements say otherwise.
- Reserve large display text for true page-level headings.
- Keep form labels, helper text and table content compact and readable.

## Spacing

- Prefer consistent spacing steps: 4, 8, 12, 16, 24, 32 and 48 pixels.
- Use tighter spacing inside dense controls and larger spacing between screen sections.

## Radius

- Use small radii for application UI.
- Cards and panels should usually stay at 8px or below.

## Motion

- Use motion only to clarify state changes.
- Respect reduced-motion preferences for non-essential animation.
